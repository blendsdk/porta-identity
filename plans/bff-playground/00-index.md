# BFF + M2M Playground Implementation Plan

> **Feature**: Server-side OIDC playground using openid-client with BFF pattern + M2M demo
> **Status**: Planning Complete
> **Created**: 2026-04-12

## Overview

The existing Porta playground is a browser-side SPA that uses `oidc-client-ts` to perform Authorization Code + PKCE flows directly from the browser. While effective for testing public client flows, it doesn't exercise the **confidential client** code paths that real SaaS applications use — where a backend server acts as the OIDC Relying Party on behalf of the user.

This plan creates a **BFF (Backend-for-Frontend) playground** — a Koa web application on port 4001 that uses `openid-client` v6 (by panva, same author as `node-oidc-provider`) to handle all OIDC flows server-side. Tokens are stored in Redis-backed server sessions; the browser only receives a session cookie. Additionally, an **M2M (Machine-to-Machine) demo** section demonstrates the `client_credentials` grant for service-to-service authentication.

This playground serves as both a developer reference implementation and a bug-finding tool — the existing SPA playground already uncovered several critical bugs in Porta, and the BFF playground will exercise entirely different code paths (confidential client auth, server-side token exchange, server-side refresh, introspection).

## Quick Reference

### Architecture

```
Browser ←→ BFF (port 4001) ←→ Porta (port 3000)
                │
                └── Redis (session store, shared with Porta)
```

### Key Flows

```
BFF Authorization Code:
  Browser → GET /auth/login → 302 to Porta → user authenticates →
  302 to /auth/callback → BFF exchanges code server-side →
  stores tokens in session → 302 to / (dashboard)

M2M Client Credentials:
  Browser → POST /m2m/token → BFF calls Porta's token endpoint
  with client_credentials → returns decoded token to page
```

### Key Decisions

| Decision             | Outcome                   |
| -------------------- | ------------------------- |
| OIDC client library  | `openid-client` v6        |
| Server framework     | Koa (matches Porta)       |
| Session store        | Redis via ioredis          |
| Template engine      | Handlebars                 |
| PKCE on BFF          | Yes (Porta requires it)    |
| Config format        | JSON (server-side)         |
| M2M client           | Separate from BFF client   |
| Port                 | 4001                       |
