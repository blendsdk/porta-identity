## Special Rules (Project-Specific)

- **Koa, NOT Express** — required for node-oidc-provider compatibility
- **Yarn Classic 1.22** — enforced via .npmrc `engine-strict=true`
- **Graceful shutdown** — SIGTERM/SIGINT → close server → disconnect DB → disconnect Redis
- **Health check** — `GET /health` must check server + DB + Redis
- **Config fail-fast** — missing/invalid env vars must cause immediate exit with clear error
- **Logger before config** — logger uses process.env directly (no config dependency)
- **12 Requirements Documents** — RD-01 through RD-12 in `requirements/`
- **Hybrid OIDC adapters** — Redis for short-lived (Session, Interaction, AuthorizationCode, ReplayDetection, ClientCredentials, PushedAuthorizationRequest), PostgreSQL for long-lived (AccessToken, RefreshToken, Grant)
- **ES256 signing** — ECDSA P-256 for token signing, keys stored as PEM in DB
- **Path-based multi-tenancy** — `/:orgSlug/*` prefix for all OIDC endpoints
- **System config service** — reads TTLs from `system_config` table with 60s in-memory cache
